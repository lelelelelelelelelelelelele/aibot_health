### 🚀 通用提取指令 (General Extraction Prompts)

请将这些 Prompt 依次输入 NotebookLM。它们不包含任何具体页码或特定产品名，而是基于**逻辑定义**来抓取信息。

#### 1️⃣ 功能：提取“核心服务体系”
**适用语料**：服务手册、价格单、设备说明书。
**指令逻辑**：识别所有“服务项目”，提取其属性（做什么、用什么、多少钱）。

```text
请扮演“服务产品经理”的角色。阅读所有上传的资料，提取所有对外提供的“服务项目”信息。

请扫描文中出现的列表、表格或段落，识别出任何符合“服务/套餐/检测”定义的实体。对于每一个识别出的服务，请尽可能提取以下维度（若文中未提及某维度则跳过）：

1. **服务标准名称**：提取最正式的全称。
2. **服务定义/流程**：简述该服务具体包含哪些步骤，或针对身体哪个部位。
3. **技术/设备支撑**：该服务是否依赖特定的仪器、AI设备或工具？
4. **时长与频次**：单次服务的时间，或建议的疗程频次。
5. **价格规则**：提取所有相关的价格数字（包括单价、会员价、团体价、体验价等）。
6. **适用人群/禁忌**：文中是否有提到谁适合做，或谁不能做？

请以结构化的 Markdown 列表形式输出，保留原文中的具体数值，不要进行模糊概括。
```

#### 2️⃣ 功能：提取“实物产品与课程”
**适用语料**：产品宣传册、配方表、课程介绍、商城截图。
**指令逻辑**：识别“可售卖实体”或“教学内容”，提取其功效和属性。

```text
请扮演“产品库管理员”的角色。阅读资料，筛选出所有“实物产品”（如茶饮、膏方、器械）和“课程/活动”（如DIY工坊）。

请按以下逻辑进行归纳：

1. **产品/课程名称**：提取准确名称。
2. **核心卖点/功效**：该产品主要解决什么问题？（如助眠、祛湿、肩颈放松）。
3. **主要成分/参数**：对于食品/药品提取主要成分；对于器械提取核心参数。
4. **使用/参与方式**：怎么吃、怎么用、或者怎么上课。
5. **规格与价格**：提取包装规格（如盒/袋）和对应价格信息。

请忽略纯修饰性的广告语，只提取具有实质信息的内容。
```

#### 3️⃣ 功能：构建“症状-解决方案”图谱 (RAG 核心)
**适用语料**：临床案例、调理方案、医学科普文章、设备功效说明。
**指令逻辑**：这是最关键的。不提取产品参数，而是提取**“因果关系”**——即“什么病用什么治”。

```text
请扮演“健康咨询顾问”的角色。阅读所有资料，构建一个“健康问题解决方案知识库”。

请扫描文中提到的各种**健康困扰、症状、亚健康状态或慢性病**（例如：失眠、高血压、脊柱侧弯、湿气重等）。
对于每一个识别出的“问题”，请总结文档中给出的所有“对策”：

1. **问题标签**：症状或疾病名称。
2. **AI/设备干预方案**：文中推荐用什么设备或技术来改善此问题？
3. **中医/物理干预方案**：推荐什么穴位、手法或经络疏通？
4. **饮食/内服建议**：推荐搭配什么茶饮或膏方？
5. **生活方式建议**：推荐什么运动或日常习惯？
6. **数据支撑**：文中是否有提及针对此问题的见效周期或有效率数据？

请以“问题 -> 综合对策”的结构进行输出，确保能覆盖文档中提到的所有健康场景。
```

#### 4️⃣ 功能：提取“规则、权益与政策”
**适用语料**：会员手册、工会合作协议、促销海报、运营SOP。
**指令逻辑**：识别逻辑规则，而非实体。

```text
请扮演“运营总监”的角色。阅读资料，提取所有关于“规则”、“权益”和“合作模式”的信息。

请重点关注并梳理以下几类逻辑：
1. **会员体系**：有哪些等级？不同等级的门槛是什么？享受什么具体权益（折扣、免费项目）？
2. **企业/B端合作**：团体采购有哪些标准？折扣阶梯是怎样的？积分如何兑换？
3. **服务保障**：是否有退改规则、预约限制或其他服务承诺？

请用清晰的条目列出这些规则，重点保留数字（折扣率、金额、人数限制）。
```

#### 5️⃣ 功能：提取“运营概况与背书”
**适用语料**：公司介绍、专家简历、联系卡片、门店SOP。
**指令逻辑**：提取静态的“元数据”。

```text
请阅读资料，提取关于“机构背景”和“运营信息”的基础数据：

1. **服务流程**：用户到店的标准接待流程是怎样的？
2. **专业资质**：提取文中提到的专家团队介绍、技术专利、或获得的荣誉认证（作为信任背书）。
3. **联系与地址**：所有的物理地址、联系电话、营业时间等信息。

请将这些分散的信息汇总成一段简明的机构介绍。
```

### 💡 为什么这么改更好？

1.  **容错率高**：无论你后续上传的是 Excel 表格还是文字版 Word，只要里面有“价格”和“服务名”，Prompt 1 都能抓到。
2.  **不遗漏细节**：之前的 Prompt 限定了页码，可能会漏掉散落在其他页的零星信息。现在的 Prompt 是**全篇扫描**。
3.  **适配 RAG 切片**：这些 Prompt 输出的结构（Markdown 列表）天然适合进行语义切片，也方便后续让大模型将其转化为 JSON。

**下一步建议**：
你可以先把这 5 个 Prompt 扔给 NotebookLM 跑一遍你现在的 PDF，看看提取出来的**Markdown 文本**质量如何。如果质量达标，我们再进行“转 JSON”的步骤。
